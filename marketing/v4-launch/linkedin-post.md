# LinkedIn-Post zur SCC v4.0.0 Release

## Variante 1 — kompakt (empfohlen)

**Das BSI hat heute C3A v1.0 veröffentlicht — 30 prüfbare Kriterien für Cloud-Souveränität. Wir haben sie direkt in unser Bewertungstool integriert.**

Der **Sovereign Cloud Compass v4.0.0** ist live: jede Souveränitäts-Bewertung beruht jetzt auf den BSI-C3A-Kriterien plus belastbaren Quellen — keine Experten-Schätzungen mehr.

Was C3A ändert:
✅ 30 prüfbare Einzelkriterien (SOV-1…6) statt holistischer Einschätzungen
✅ C1- vs. C2-Varianten für EU-Bezug und deutschen Bezug
✅ Audit-Tauglichkeit — Compliance-Teams können einzelne Kriterien gegen ihre Anforderungen prüfen

Im SCC neu:
🔹 **C1/C2 Audit-Strenge-Toggle** — Provider mit nur EU-Erfüllung werden im DE-Modus konsequent heruntergestuft, exakt wie es das BSI vorsieht
🔹 **SOV-7 Compliance-Katalog** (10 Kriterien) — schließt die Sicherheitslücke, die das BSI bewusst auf C5/IT-Grundschutz auslagert
🔹 **16 Cloud-Provider** mit Quellenbelegen — von AWS bis STACKIT, neu auch Oracle EU Sovereign Cloud
🔹 **Spider-Visualisierung** je Provider mit vollem C3A-Profil

Werte-Verschiebungen, die ein anderes Bild zeichnen:
↗️ AWS European Sovereign Cloud: SEAL-2 → SEAL-3
↗️ Microsoft DELOS Cloud: SEAL-2 → SEAL-3
↘️ AWS Outpost: SEAL-1 → SEAL-0

Warum jetzt? Der angekündigte **EU Cloud and AI Development Act (CADA)** könnte C3A-Teile im Mai 2026 in verbindliche Rechtsnormen überführen. Wer heute KRITIS-, Bundes- oder NIS2-Beschaffung plant, sollte die Kriterien jetzt schon kennen.

## Variante 2 — etwas länger, mehr methodischer Kontext

**Cloud-Souveränität wird ab heute auditierbar.**

Das BSI hat am 27.04.2026 die *Criteria enabling Cloud Computing Autonomy (C3A) v1.0* veröffentlicht — und damit eine klare Methodik geliefert: 30 prüfbare Kriterien, die das EU Cloud Sovereignty Framework von einer Klassifizierung zu einem **Audit-Werkzeug** machen.

Wir haben das direkt in den Sovereign Cloud Compass integriert. Heute live als v4.0.0:

🔹 **30 BSI-C3A-Kriterien** je Cloud-Provider mit Quellenbeleg
🔹 **SOV-7 Compliance-Katalog** für die Sicherheitsaspekte (10 Kriterien orientiert an ISO 27001, BSI C5/IT-Grundschutz, ISO 27017/27018/27701, SOC 2, KRITIS, NIS2)
🔹 **C1/C2 Audit-Strenge-Toggle** — Wechsel zwischen EU- und deutschem Bezug genau so, wie BSI C3A es methodisch vorsieht
🔹 **16 Provider** unter der Lupe: AWS, Azure, GCP, Oracle, AWS European Sovereign Cloud, Microsoft DELOS, **Oracle EU Sovereign Cloud (neu)**, STACKIT, IONOS, T Cloud Public, SAP Cloud Infrastructure, OpenStack, VMware, Google Dedicated Cloud, Azure Stack HCI, AWS Outpost

Mit der Umstellung verschieben sich Werte messbar — und realitätsnäher. AWS ESC und DELOS steigen auf SEAL-3, AWS Outpost fällt auf SEAL-0, weil die US-Lieferkette konsequent gewertet wird.

Was das praktisch bedeutet: Wer heute Cloud-Beschaffung für **KRITIS, Bundesverwaltung oder NIS2** plant, kann jede Bewertung gegen die BSI-Kriterien prüfen — mit Quellenbeleg pro Aussage. Wenn der angekündigte **EU Cloud and AI Development Act (CADA)** im Mai 2026 Teile von C3A in verbindliche Rechtsnormen überführt, ist die Datenbasis bereits da.
